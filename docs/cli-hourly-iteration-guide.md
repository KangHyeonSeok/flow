# CLI 시간 단위 개발 반복 지침

이 문서는 Flow 저장소에서 CLI 에이전트를 매 시간 반복 실행할 때 사용할 운영 규칙을 정리한다. 목표는 `문서 정합성 유지`, `문서 기반 우선순위 정리`, `바로 구현`, `다음 반복을 위한 학습 축적` 네 가지다.

## 1. 반복의 기본 구조

한 번의 반복은 항상 두 단계로 끝낸다.

1. 문서와 구현을 비교해 문서를 업데이트하고, 이번 반복의 할 일을 문서에 정리한다.
2. 방금 정리한 할 일을 기준으로 직접 구현하고 검증한다.

반복 종료 직전에는 중요한 내용을 `CLAUDE.md`에 남겨 다음 반복의 시작 비용을 줄인다.

## 2. 시작 체크리스트

반복 시작 시 에이전트는 아래를 먼저 확인한다.

- 현재 작업 대상 spec 또는 기능 문서
- 관련 설계 문서와 프로젝트 문서
- 최근 구현 변경 파일과 테스트 파일
- 직전 반복에서 남긴 `CLAUDE.md`의 최신 노트
- 열려 있는 실패, 미완료 항목, 임시 우회 로직

이 단계에서 해야 할 판단은 아래다.

- 문서가 구현보다 뒤처졌는가
- 구현이 문서 의도에서 벗어났는가
- 이번 한 시간 안에 끝낼 수 있는 단위로 작업이 쪼개져 있는가

## 3. 1단계: 문서 업데이트와 할 일 정리

구현에 손대기 전에 먼저 문서를 맞춘다.

### 해야 하는 일

- 문서와 실제 동작이 다르면 문서를 수정한다.
- 설계가 바뀌었다면 관련 설계 문서 또는 spec 문서에 반영한다.
- 이번 반복의 작업 목록을 문서에 남긴다.

### 작업 목록 작성 규칙

작업 목록은 채팅이 아니라 저장소 문서에 남긴다. 형식은 아래 중 하나를 사용한다.

- 대상 문서 안의 `Now / Next / Later`
- 대상 문서 안의 `이번 반복 할 일`
- 별도 작업 문서의 체크리스트

각 항목에는 아래 정보가 있어야 한다.

- 목적
- 변경 대상 파일 또는 기능
- 완료 판단 기준

### 권장 템플릿

```md
## 이번 반복 할 일

### Now

- [ ] 문서와 현재 구현 차이를 반영한다.
  완료 기준: 관련 문서가 현재 코드 구조와 상태를 설명한다.

- [ ] 가장 중요한 구현 1건을 완료한다.
  완료 기준: 관련 테스트 또는 수동 검증 결과가 남아 있다.

### Next

- [ ] 이번 반복 다음으로 이어질 후속 작업을 1~3개 정리한다.

### Later

- [ ] 지금 바로 하지 않을 항목을 보류 이유와 함께 남긴다.
```

## 4. 2단계: 문서 기준 구현

구현은 반드시 방금 정리한 `Now` 목록을 기준으로 진행한다.

### 실행 규칙

- 구현 범위가 커지면 먼저 문서의 작업을 더 작은 단위로 다시 쪼갠다.
- 문서에 없는 새 작업이 생기면 먼저 문서에 추가한 뒤 구현한다.
- 코드만 바꾸고 문서를 그대로 두지 않는다.
- 테스트 가능하면 테스트를, 테스트가 없으면 수동 검증 절차를 남긴다.

### 반복 안에서 남겨야 하는 결과

- 실제 코드 변경
- 검증 결과
- 남은 리스크와 후속 작업

## 5. 반복 종료 시 `CLAUDE.md` 업데이트

매 반복 종료 전에 `CLAUDE.md`를 갱신한다. 이 파일은 다음 반복을 위한 압축 메모리다.

### 반드시 기록할 것

- 이번 반복의 범위
- 완료한 작업
- 다음 반복의 첫 작업
- 막힌 원인 또는 남은 리스크
- 새로 확인한 제약, 함정, 결정 사항
- 실행한 테스트와 검증 누락 항목

### 기록 원칙

- 짧게 쓴다.
- 사실만 남긴다.
- 다음 반복이 즉시 행동할 수 있게 쓴다.
- 장기 설계 설명은 관련 문서에 남기고 `CLAUDE.md`에는 요약만 남긴다.

## 6. 권장 산출물 위치

- 기능 의도와 상태: 관련 spec 문서 또는 설계 문서
- 프로젝트 수준 운영 규칙: `docs/flow-project-document.md` 또는 관련 운영 문서
- 시간 단위 반복 학습: `CLAUDE.md`

즉 `CLAUDE.md`는 로그 저장소가 아니라 `다음 반복의 부팅 메모리`로 사용한다.

## 7. 매 반복 시작 프롬프트 예시

```text
관련 문서, 구현, 테스트를 먼저 확인해서 문서를 최신 상태로 맞추고 이번 반복의 할 일을 문서에 정리해. 그 다음 정리한 할 일 중 가장 중요한 것부터 직접 구현하고 검증해. 반복이 끝나면 완료한 것, 남은 것, 다음 작업, 리스크, 검증 결과를 CLAUDE.md에 업데이트해.
```

## 8. 좋은 반복의 기준

좋은 반복은 아래 조건을 만족한다.

- 시작 10분 안에 문서와 코드의 불일치가 줄어든다.
- 구현 작업이 문서의 `Now` 항목과 정확히 연결된다.
- 반복 종료 시 다음 반복의 첫 작업이 명확하다.
- 같은 실수를 피하기 위한 학습이 `CLAUDE.md`에 누적된다.
